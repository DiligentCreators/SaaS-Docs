# Calendar

The Calendar module shows timed work from the Scheduling Platform in month, week, day, and agenda views.

## What you can do

- Switch between **Month**, **Week**, **Day**, and **Agenda**
- Navigate with previous / today / next
- Filter by calendar and category
- Set colors and default view preferences
- Create and edit **calendar events** (stored on the Scheduling Platform)
- View items published by other modules (for example Tasks) as read-only on the calendar

## Calendars

- **Personal** — created automatically for you; cannot be deleted
- **Team** — shared calendars with owner / editor / viewer members (managed by users with `calendar.manage`)

## Categories and colors

Categories are workspace labels for organizing events. Event color comes from the event override, category, schedule item, or calendar default.

## Requirements

Your workspace must have the **Calendar** module installed, and you need `calendar.view` (plus create/update permissions for editing).
