# Tasks — User Guide

## Who can use Tasks

Your workspace must have the **Tasks** module installed (included by default on new workspaces). Your role must include the relevant permissions (view, create, update, delete, assign, complete).

## List & filters

Open **Tasks** from the sidebar.

- Search by title or description
- Filter by status (open / in progress / completed / cancelled) and priority
- Sort and paginate the table

## Create & edit

1. Click **New task**
2. Enter title (required) and optional description, status, priority, due date, and assignee
3. Save

Edit from the row menu or the detail drawer.

## Assignment

Users with **assign** permission can set or clear the assignee from the detail drawer. The assignee receives an email notification when someone else assigns them.

## Complete & reopen

Users with **complete** permission can mark a task completed (sets `completed_at`) or reopen it from the detail drawer.

## Notes

The **Notes** tab holds free-form progress history on the task.

## Activity timeline

The **Activity** tab shows create, update, assignment, complete, reopen, note, and delete-related events.
