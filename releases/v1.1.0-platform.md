This document is the official **Platform Foundation** release record. All subsequent product work belongs to business modules (CRM Sprint 2+, Sales, Billing modules, ERP, etc.), not platform redesign.

---

## Overview

Diligent Creators SaaS Platform v1.1.0 freezes the shared multi-tenant foundation:

- **Backend:** Laravel 13, Sanctum, Spatie Permission, Cashier (Stripe)
- **Frontend:** React 19 SPA (Central admin + Tenant application) with shared AppLayout
- **Docs:** Architecture, auth, billing, settings, RBAC, module standard, testing guides

Verified green: Pest **230/230**, Playwright **69/69**, manual browser QA — production-ready.

---

## Architecture Summary

| Layer | Responsibility |
|-------|----------------|
| Authentication | Central + Tenant login/register, email verification, password reset, remember-me, lockout |
| Tenancy | Workspace isolation, domains, provisioning, entitlements |
| Authorization | Module licensing (`module:{slug}`) + Spatie RBAC (`can:{permission}`) |
| Marketplace | Module catalog, install/cancel/deactivate, default-included Leads/Tasks |
| Billing | Gateway-agnostic invoices/payments, consolidated billing, Stripe + Manual drivers |
| Settings | Central → Tenant → system hierarchy (branding, mail, locale, currency) |
| Security | Audit logs, impersonation sessions, suspended users, lockout |
| Reference modules | **Leads**, **Tasks** (blueprint for all future modules) |

See [architecture/platform-freeze.md](../architecture/platform-freeze.md).

## Database Changes

Identity, RBAC, module catalog/subscriptions, billing ledger, payment gateways, settings, activity/impersonation, Leads, and Tasks tables as documented in [architecture/database.md](../architecture/database.md).

**v1.1.0 hardening:** billing webhook tenant resolution via `payment_id` metadata fallback; auth lockout messaging; expanded validation/security tests. No greenfield schema redesign.

---

## Breaking Changes

Relative to the pre-marketplace / plan-based licensing era:

1. Plans / Features / usage limits removed — licensing is module subscriptions only.
2. Central SPA routes under `/central/*`.
3. Entitlements API shape `{ core, modules }`.

Within the modular-marketplace line, v1.1.0 is a hardening release (no additional intended API breaks).
